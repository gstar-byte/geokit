# 项目方向：免费在线工具站

> 基于 TrustMRR 数据 + 竞品调研，2026年7月
> 最后更新：2026-07-15

## 核心策略

**做免费在线工具站，不做内容站/目录站。**

- 工具站不依赖内容质量，用户来用工具，不是来读文章
- 工具天然吸引外链（博主写教程时引用、用户在论坛分享）
- 每个工具 = 一个 SEO 着陆页，精准匹配搜索意图
- 纯前端工具零服务器成本，有 API 成本的工具可小成本试水

### 为什么不做目录站/内容站
- AI 生成内容质量存疑，Google 越来越能识别，新站不排名
- 手动写深度内容不现实（无开发经验 + 精力有限）
- 目录站需要大量内容才能做深，内容薄了 Google 不排名
- 已有项目验证：CADGuide.tools（AI生成内容）上线2个月 Google 收录但不排名

## 方向：GEOKit — GEO 工具包

### 什么是 GEO
GEO = Generative Engine Optimization（AI 搜索引擎优化），2026 年最热门的 SEO 话题。
优化网站使其在 ChatGPT、Perplexity、Google AI Overviews 等 AI 搜索引擎中获得引用和曝光。

### 为什么选 GEO 工具包
- 单个工具都有竞品，但没有人把它们捆绑成一个完整的 GEO 工具包
- GEO 概念搜索量暴涨中（"llms.txt"、"AI SEO"、"GEO"）
- SEO 博主写 GEO 教程时一定会引用工具 = 天然外链
- 全部纯前端，零成本
- 和 CADGuide 的工具捆绑模式完全复用

### 6 个工具

| 工具 | 功能 | 技术 |
|---|---|---|
| llms.txt 生成器 | 生成 llms.txt 文件 | 纯前端 |
| AI Robots.txt 生成器 | 控制 AI 爬虫访问 | 纯前端 |
| Schema for AI 生成器 | 生成 AI 友好的 JSON-LD | 纯前端 |
| AI 就绪度检测器 | 检测网站对 AI 的友好度评分 | API route |
| llms.txt 验证器 | 验证 llms.txt 是否合规 | 纯前端 |
| GEO 检查清单 | GEO 优化步骤引导 | 纯前端 |

### 技术栈
- Next.js 14 + TypeScript + Tailwind CSS
- Vercel 部署
- 纯前端工具 + 1 个 API route（就绪度检测）

### 变现路径
1. 免费工具引流 → 付费功能变现（批量检测、监控、API）
2. 联盟佣金（推荐 Surfer SEO、GetMentioned 等 GEO 付费工具）
3. 广告收入（流量起来后）

### SEO 策略
- 每个工具页面 = 一个精准搜索词着陆页
- 工具天然吸引外链：SEO 博主写 GEO 教程时引用
- 每周花 1-2 小时在 Reddit/论坛分享工具

---

## 关键原则

1. **工具优先，内容其次** — 工具本身是核心价值，不需要大量文章
2. **纯前端优先** — 先做零成本的浏览器端工具
3. **每个工具一个搜索词** — 工具页面精准匹配用户搜索意图
4. **必须做外链** — 每周 1-2 小时在社区分享工具
5. **不要同时做多个站** — 集中精力到一个项目
